# i18n — 새 문자열 / 새 locale 추가하기

Tokenova는 frontend 사전 + backend 사전 두 곳에서 KO/EN을 관리합니다. 추가는 두 곳 모두 동기 필요.

## 빠른 흐름

```
새 UI 문자열 추가 시:
  1. src/i18n/locales/ko.ts에 키 + 한국어 추가
  2. src/i18n/locales/en.ts에 같은 키 + 영어 추가  (TS 컴파일러가 누락 알려줌)
  3. 코드에서 t("nav.your_key") 호출
  4. (HTML 정적 문자열인 경우) <span data-i18n="nav.your_key">기본값</span>
```

## Frontend 사전 구조

`src/i18n/locales/ko.ts`가 master. 그 객체의 TypeScript 타입이 자동으로 영어 사전의 contract가 됨:

```typescript
// ko.ts
export const ko = {
  nav: {
    today: "오늘",
    codex: "도감",
    // ... 새 키 추가:
    settings: "설정",
  },
  // ...
};

// en.ts
import type { Dictionary } from "..";
export const en: Dictionary = {
  nav: {
    today: "Today",
    codex: "Codex",
    settings: "Settings",  // ← 빠뜨리면 컴파일 에러
  },
  // ...
};
```

타입 일치는 `npx tsc --noEmit`로 항상 확인. CI도 이 check를 돕니다.

## t() 함수

```typescript
import { t } from "../i18n";

// 정적 문자열
t("nav.today")                        // → "오늘" 또는 "Today"

// {placeholder} 치환
t("today.draw_bar.connected_n", { count: 3 })
// → "별 3개 연결됨" 또는 "3 stars connected"
```

placeholder는 `{name}` 형식. params가 없거나 누락된 key가 있으면 그대로 둠 (없는 경우 placeholder 텍스트 보임 — 빨리 발견됨).

## HTML 정적 문자열

`data-i18n` attribute로 마킹하면 `applyDomI18n()`이 채워줌:

```html
<button data-i18n="today.draw_bar.cancel">취소</button>
<input data-i18n-attr-placeholder="today.cluster_name_placeholder" />
<span data-i18n-attr-title="today.tokens_label">토큰</span>
```

원본 텍스트는 fallback 역할 — applyDomI18n이 init과 locale 변경 시마다 덮어씀.

## 동적 + 인터폴레이션

JS에서 직접 textContent/innerHTML 설정 시:

```typescript
const $label = document.getElementById("draw-bar-label");
if ($label) $label.innerHTML = t("today.draw_bar.connected_n", { count });
```

## Backend (Rust) 사전

알림 / 트레이 텍스트 / 백엔드 직접 emit 문자열은 `src-tauri/src/i18n.rs`에 별도:

```rust
pub fn planet_rarity_title(locale: Locale, rarity: Rarity) -> &'static str {
    match (locale, rarity) {
        (Locale::Ko, Rarity::Mythic) => "신화 행성 발견!",
        (Locale::Ko, Rarity::Legendary) => "전설 행성 발견!",
        // ...
        (Locale::En, Rarity::Mythic) => "Mythic planet discovered!",
        (Locale::En, Rarity::Legendary) => "Legendary planet discovered!",
        // ...
    }
}
```

새 알림 메시지 추가 시:
1. Rust 함수 시그니처에 새 case 추가 (KO + EN 둘 다)
2. 호출하는 곳에서 `let locale = crate::i18n::current_locale(&self.db);` 로 현재 locale 얻고 함수 호출

`current_locale`은 DB의 `settings` 테이블에서 `locale` 키를 매 호출마다 lookup — 알림은 드물어서 캐싱 안 함.

## locale 상태 + 변경

```typescript
import { getLocale, setLocale, subscribeLocale } from "../i18n";

getLocale()           // "ko" | "en"
await setLocale("en") // persist + applyDomI18n + 모든 subscriber 호출

subscribeLocale((newLocale) => {
  // 이 view가 dynamic 문자열을 다시 그려야 함
  refresh();
});
```

폴링 기반 view (예: today.ts의 3초 poll)는 자동으로 다음 poll에서 새 locale 반영. 즉시 반영이 필요한 view (예: settings.ts의 update banner 상태)는 `subscribeLocale`로 listen.

## 새 locale 추가하기 (예: 일본어)

1. **타입 확장**: `src/i18n/index.ts`
   ```typescript
   export type Locale = "ko" | "en" | "ja";
   ```
2. **사전 파일 생성**: `src/i18n/locales/ja.ts` — `en.ts`를 복사해서 시작, 모든 값 일본어로 교체. `Dictionary` 타입이 누락 키 자동 alert.
3. **DICTS 맵 등록**: `src/i18n/index.ts`
   ```typescript
   import { ja } from "./locales/ja";
   const DICTS: Record<Locale, Dictionary> = { ko, en, ja: ja as Dictionary };
   ```
4. **persistence validation 확장**: `src-tauri/src/commands.rs::set_locale`
   ```rust
   if value != "ko" && value != "en" && value != "ja" {
       return Err(format!("unsupported locale: {value}"));
   }
   ```
5. **detectSystemLocale 확장**: `src/i18n/index.ts`
   ```typescript
   if (l.toLowerCase().startsWith("ja")) return "ja";
   ```
6. **Settings UI 버튼 추가**: `index.html`
   ```html
   <button class="settings-lang-btn" data-locale="ja" data-i18n="settings.language.ja">日本語</button>
   ```
7. **Backend i18n 확장**: `src-tauri/src/i18n.rs`의 `Locale` enum + 모든 함수의 match arm에 `Locale::Ja` 추가.

## 검증

```bash
# TS 사전 일관성
npx tsc --noEmit

# Rust 사전 일관성 (모든 enum variant이 케이스 처리됐는지)
cargo check
```

Rust는 enum의 모든 case를 매칭 안 하면 컴파일 에러 — 안전.

## 우선순위 / fallback

1. 사용자가 명시한 설정 (DB `locale` 키)
2. `navigator.languages` 휴리스틱 (시스템 OS locale)
3. 최종 fallback: 한국어 (master 사전)

키 자체가 빠진 경우:
- 현재 locale에 키 없음 → master(ko) 사전에서 시도
- ko에도 없음 → 키 string 자체를 반환 (dev에서 즉시 보임)
